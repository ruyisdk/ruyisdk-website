# 报告问题

感谢使用 RuyiSDK。如果你在使用过程中遇到了问题、发现了 Bug、有功能建议，或者希望 RuyiSDK 支持新的软件包，都欢迎提交 issue 或在论坛发帖反馈。

## 在哪里提交 issue？

RuyiSDK 的各个组件分别托管在不同的 GitHub 仓库中。请尽量根据问题类型选择对应的仓库提交 issue：

| 问题类型 | 查看所有 issue | 提交新 issue |
| --- | --- | --- |
| Ruyi 包管理器本身的问题，例如命令行行为异常、安装失败、虚拟环境、设备刷写等 | [查看 issue](https://github.com/ruyisdk/ruyi/issues) | [提交 issue](https://github.com/ruyisdk/ruyi/issues/new) |
| 软件包链接失效、软件包版本落后、希望新增软件包 | [查看 issue](https://github.com/ruyisdk/packages-index/issues) | [提交 issue](https://github.com/ruyisdk/packages-index/issues/new) |
| RuyiSDK Eclipse 插件相关问题 | [查看 issue](https://github.com/ruyisdk/ruyisdk-eclipse-plugins/issues) | [提交 issue](https://github.com/ruyisdk/ruyisdk-eclipse-plugins/issues/new) |
| RuyiSDK VS Code 扩展相关问题 | [查看 issue](https://github.com/ruyisdk/ruyisdk-vscode-extension/issues) | [提交 issue](https://github.com/ruyisdk/ruyisdk-vscode-extension/issues/new) |
| ruyisdk.org 网站前端相关问题 | [查看 issue](https://github.com/ruyisdk/ruyisdk-website/issues) | [提交 issue](https://github.com/ruyisdk/ruyisdk-website/issues/new) |
| api.ruyisdk.cn 后端相关问题 | [查看 issue](https://github.com/ruyisdk/ruyi-backend/issues) | [提交 issue](https://github.com/ruyisdk/ruyi-backend/issues/new) |

提交 issue 前，建议先搜索一下是否已经存在相同或类似的问题。如果已经有人报告过相同问题，可以在已有 issue 下补充你的环境信息、复现步骤或其他线索；如果只是类似问题，也可以在新 issue 中提及相关的旧 issue，方便开发者关联排查。

## 提交 issue 时需要提供哪些信息？

为了帮助开发者更快理解和定位问题，建议尽可能提供以下信息：

1. **环境信息**  
   包括出问题的软件版本、操作系统版本、硬件平台等。  
   如果问题与 Eclipse 或 VS Code 插件有关，也请提供对应 IDE 的版本信息。  
   请注意不要公开账号、密码、访问令牌、私有地址等敏感信息。

2. **问题描述**  
   说明你遇到了什么问题、你原本期望发生什么、实际发生了什么。  
   如果有报错信息、截图或日志，也可以一并附上。

3. **复现步骤**  
   说明在什么情况下会出现这个问题。  
   如果可以，请提供一个最小化、稳定可复现的操作流程。

## 没有 GitHub 账号怎么办？

如果没有 GitHub 账号，或者觉得提交 issue 不方便，也可以在 RuyiSDK 论坛 [ruyisdk.cn](https://ruyisdk.cn/) 发帖反馈。开发人员和其他用户也会尽力帮助你定位和解决问题。
